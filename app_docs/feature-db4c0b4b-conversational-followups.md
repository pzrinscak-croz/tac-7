# Conversational Follow-ups

**ADW ID:** 26
**Date:** 2026-03-27
**Specification:** specs/issue-db4c0b4b-adw-26-sdlc_planner-add-conversational-followups.md

## Overview

Adds conversational follow-up support to the Natural Language SQL Interface. After running a query, the next query automatically includes the previous question and generated SQL as context, allowing the LLM to handle follow-ups like "now filter that by city" or "show that as percentages instead." A "Clear context" button resets to standalone mode.

## What Was Built

- Client-side conversation context tracking (previous query + SQL pair)
- Automatic context injection into LLM prompts for follow-up queries
- "Continuing from: '{query}'" UI indicator when context is active
- "Clear context" button to reset to standalone mode
- Automatic context clearing on failed queries, file uploads, and table removals
- Server-side `build_conversation_context()` helper for prompt augmentation
- Unit tests for conversation context across both OpenAI and Anthropic providers
- E2E test command for conversational follow-up flows

## Technical Implementation

### Files Modified

- `app/server/core/data_models.py`: Added optional `previous_query` and `previous_sql` fields to `QueryRequest`
- `app/server/core/llm_processor.py`: Added `build_conversation_context()` helper; updated `generate_sql_with_openai()`, `generate_sql_with_anthropic()`, and `generate_sql()` to accept and inject conversation context into prompts
- `app/client/src/types.d.ts`: Added `previous_query?` and `previous_sql?` to `QueryRequest` interface
- `app/client/src/main.ts`: Added conversation state variables, `clearConversationContext()` and `updateContextIndicator()` helpers, context sending in query flow, context clearing on upload/remove
- `app/client/index.html`: Added `#context-indicator` div with label and clear button
- `app/client/src/style.css`: Added styles for `.context-indicator`, `.context-label`, and `.clear-context-button`
- `app/server/tests/core/test_llm_processor.py`: Added `TestConversationContext` class with 5 tests covering context building, prompt injection for both providers, backward compatibility, and routing
- `.claude/commands/e2e/test_conversational_followups.md`: New E2E test for the full conversational flow

### Key Changes

- Context is tracked entirely client-side (no server session state), keeping the server stateless
- Only the most recent successful query/SQL pair is stored (single-turn context, not multi-turn history)
- Both `previous_query` and `previous_sql` must be present for context to be injected; if either is missing, the query runs in standalone mode
- The context block is prepended to the LLM prompt before the current query text, instructing the model to resolve references like "that", "those results", etc.
- Failed queries do not update context, preserving the last successful query as reference

## How to Use

1. Upload a CSV file or use sample data to populate the database
2. Enter a natural language query (e.g., "Show me all users") and submit
3. After results appear, notice the "Continuing from: 'Show me all users'" indicator below the query input
4. Enter a follow-up query referencing the previous results (e.g., "filter that by city = 'New York'")
5. The LLM uses the previous context to generate correct SQL without re-specifying tables or conditions
6. Click "Clear context" to reset and start a fresh standalone query

## Configuration

No additional configuration required. The feature uses existing LLM provider settings and API keys.

## Testing

- **Unit tests**: `cd app/server && source .venv/bin/activate && uv run pytest` — includes `TestConversationContext` class with tests for context building, prompt injection (OpenAI + Anthropic), backward compatibility, and context routing
- **E2E test**: Run `/e2e:test_conversational_followups` to validate the full conversational flow in the browser
- **Type check**: `cd app/client && bun tsc --noEmit`
- **Build**: `cd app/client && bun run build`

## Notes

- Context is limited to one turn of history. Multi-turn history could be a future enhancement but adds prompt length concerns.
- The prompt injection of previous SQL is safe because the SQL was generated by the LLM itself in the prior turn.
- No new libraries are needed for this feature.
- The query text in the context indicator is truncated to 50 characters with ellipsis for long queries.
